System Design



Seamless Experience - User-Flow Led By Intuition

The most important aspect in designing an accessibility-driven app was crafting something that is led by intuition rather than the set of mechanical instruction the user needs to follow. The approach was to not think like a developer, but like an every-day user, who would use this app extensively. We asked ourselves and others, what would be the most natural way to use a fitness tracker? That led us to the following feedback-oriented design and sequence diagrams, where each element of the user-flow was carefully chosen and implemented into a cohesive structure we have today.


Sequence Diagrams (For system design page)

Sequence diagrams will be broken down and explained in multiple segments for better understanding and overall comprehension. They will cover the following main points:

  • Onboarding

  • Workouts

    • New Workouts

    • Saved Workouts

    • Active Workouts

  • Insights


On boarding

Sensor Integration

First thing the user needs to do, in order to use this app to the full extent of its capabilities, is simply scanning and selecting the appropriate Bluetooth sensor from the settings page. Although simple and intuitive from user point of view, it required multiple processes in the background in order to achieve this.


Workouts


New Workouts
Creating New Workouts For Fast (Re)use

Seamless creation of a new workout. The user just needs to set the workout name. select activity type and training intent using two universally-accessible multi-option buttons with Text-To-Speech integration.

There are just 2 pathways from this point, either saving the workout for later use, or following the stream-lined navigation pattern and starting the workout, which will be automatically saved in the process.


Saving New Workout

In the case the user wants to setup the workouts prior to the exercise, it can seamlessly saved them by a click of a button.
